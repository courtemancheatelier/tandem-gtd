# tandem-email-inbound

Cloudflare Email Worker that delivers inbound email into a Tandem instance's
inbox via the `/api/webhooks/email-inbound` endpoint. Supports multiple
Tandem instances (e.g., beta + prod) on a single zone using local-part
prefix dispatch (`{prefix}+{token}@your-domain.com`).

## One-time setup

### 1. Cloudflare Email Routing

On your zone (`your-domain.com`) in the Cloudflare dashboard:

- **Email → Email Routing**: enable if not already enabled. Cloudflare adds
  MX records on your zone apex automatically.
- **Email → Email Routing → Settings → Subaddressing**: enable.
  This lets `inbox-beta+anything@your-domain.com` route through the
  `inbox-beta@your-domain.com` base address.
- **Email → Email Routing → Destination addresses**: add the fallback
  address you'll use for `FALLBACK_FORWARD` and verify it (Cloudflare
  emails a confirmation token).

### 2. Per-Tandem-instance env

On each Tandem server (e.g., `/opt/tandem-beta/.env`):

```
EMAIL_WEBHOOK_SECRET=<generate with: openssl rand -hex 32>
EMAIL_INBOX_DOMAIN=your-domain.com
EMAIL_INBOX_LOCAL_PREFIX=inbox-beta+       # match ROUTE_LOCAL_BETA + "+"
```

Restart the service (`sudo systemctl restart tandem-beta`).

### 3. Worker config

```bash
cd cloudflare-worker
cp wrangler.toml.example wrangler.toml     # edit values for your domain + fallback
npm install
npx wrangler login
npx wrangler deploy
```

Then set the secrets (once per instance):

```bash
# BETA (must match EMAIL_WEBHOOK_SECRET in /opt/tandem-beta/.env)
printf '<your-beta-secret>' | npx wrangler secret put WEBHOOK_SECRET_BETA

# PROD (when you're ready)
printf '<your-prod-secret>' | npx wrangler secret put WEBHOOK_SECRET_PROD
```

The Worker will upload with `No deploy targets` — that's correct for an
Email-Routing-only Worker. It runs whenever Cloudflare's Email Routing
invokes it via the zone catch-all or a custom address rule, not via HTTP.

### 4. Zone catch-all route

**Email → Email Routing → Routing rules → Catch-all address**:

- Action: **Send to a Worker**
- Destination: `tandem-email-inbound`
- Save

The Worker then dispatches based on local-part prefix:

```
inbox-beta+<token>@your-domain.com → beta Tandem webhook
inbox-prod+<token>@your-domain.com → prod Tandem webhook
<anything-else>@your-domain.com    → forwarded to FALLBACK_FORWARD
```

## Verification

In your Tandem instance, go to **Settings → Email Capture** and enable it.
You'll get a personal address like `inbox-beta+abc123@your-domain.com`.
Send a test email to it from any account; it should appear in your Tandem
inbox within a few seconds with source badge "via email from ...".

If nothing arrives, check **Email → Email Routing → Overview → Sessions**
in Cloudflare. The `Result` column shows why delivery failed (dropped,
rejected, forwarded). Tail the Worker:

```bash
npx wrangler tail tandem-email-inbound --format=pretty
```

## Security notes

- `/api/webhooks/email-inbound` on the Tandem side is exempt from
  Origin/CSRF checks (server-to-server) but authenticates with the shared
  secret via constant-time comparison. Don't expose it unauthenticated.
- Token length (≥8) is validated both in the Worker and on the Tandem
  webhook side. Tokens are 128-bit URL-safe random values generated by the
  Tandem instance.
- The Worker's HTML-stripping regex is defense-in-depth only. Inbox notes
  are rendered via React (auto-escaped); do not switch to
  `dangerouslySetInnerHTML` on that field without replacing the regex with
  a proper HTML sanitizer first.
- The body/subject size caps (10,000 / 500 chars) are enforced in both
  places; don't remove them.
