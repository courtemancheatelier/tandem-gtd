# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in Tandem, please report it responsibly. **Do not open a public issue.**

**Email:** [security@tandemgtd.com](mailto:security@tandemgtd.com)

Include as much detail as possible:

- Description of the vulnerability
- Steps to reproduce
- Affected versions or commits
- Potential impact

## Response

Tandem is a solo-maintained project. There are no firm SLAs, but the maintainer will respond as quickly as reasonably possible. Critical vulnerabilities are prioritized above all other work.

## Disclosure Timeline

Please allow up to **90 days** from the initial report before publicly disclosing the vulnerability. This gives the maintainer time to develop, test, and release a fix.

If a fix is released sooner, public disclosure can happen sooner — coordinated with the reporter.

## Credit

Security reporters will be credited in release notes (with their permission). If you'd like to be credited under a specific name or handle, include that in your report.

## Scope

This policy applies to the Tandem codebase and any infrastructure operated by the project maintainer (e.g., tandemgtd.com). Self-hosted instances are the responsibility of their operators.

## Supported Versions

Security fixes are applied to the latest release. There is no backporting to older versions at this time.
