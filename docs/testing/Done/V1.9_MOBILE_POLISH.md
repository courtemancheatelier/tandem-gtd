## 4. Pull-to-Refresh | Works

**Where:** Do Now page and Inbox page, on mobile

| Step | Expected |
|------|----------|
| Scroll to top, pull down | Arrow indicator appears, rotates as you pull further |
| Pull past threshold (~80px with 0.4x resistance) | Arrow fully rotated, turns primary color |
| Release past threshold | Spinner replaces arrow, data refreshes, indicator retracts |
| Release before threshold | Indicator springs back, no refresh |
| Scrolled partway down, pull down | Normal scroll — no pull indicator activates |
| On desktop | Nothing happens (indicator is `md:hidden`, touch events don't fire) |
| PWA standalone mode | No native pull-to-refresh conflict (suppressed by `overscroll-behavior-y: contain`) |

**Quick data check:** Complete or add a task on another device/tab, then pull-to-refresh — the change should appear.

## 5. Swipe-to-Complete | Works

**Where:** Any task card on Do Now page, on mobile

| Step | Expected |
|------|----------|
| Swipe right on a task card | Green background revealed behind card with a check icon |
| Swipe past threshold (~120px) | Haptic buzz (25ms on Android), check icon scales up |
| Release past threshold | Card animates off-screen to the right, task completes, undo toast appears |
| Tap undo in toast | Task restored |
| Swipe right but release before threshold | Card springs back to original position |
| Swipe left | Nothing happens (right-only) |
| Scroll vertically through task list | No horizontal swipe interference — direction locks after 10px |
| Tap a task card | Expand/collapse still works normally |
| Tap status circle | Still works — haptic + complete (for in-progress tasks) |
| Swipe on an already-completing task | Disabled — no swipe response |
| On desktop | No swipe behavior — `overflow-hidden` wrapper is `md:overflow-visible` |
