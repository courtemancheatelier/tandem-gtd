# Feature Visibility

Tandem includes several optional features beyond the core GTD workflow. You can hide features you don't use to keep your navigation clean and focused.

## How It Works

Feature visibility operates at two levels:

1. **Admin level** -- Your server admin controls the default state for each feature (see Admin Configuration below).
2. **User level** -- Each user can show or hide features from their own navigation, within the limits set by the admin.

## Hiding Features

1. Go to **Settings > Features**
2. Toggle off any features you don't need
3. The navigation updates immediately

- Features set to **Off** by your admin appear grayed out with a "Disabled by admin" label and cannot be enabled.
- Features set to **Off by default** appear with a hint -- toggle them on if you want to use them.

## Toggleable Features

The following features can be shown or hidden:

| Feature | Description |
|---------|-------------|
| **Dashboard** | Overview dashboard with project health, velocity, and other widgets |
| **Insights** | Analytics, trends, and behavioral data |
| **Card File** | Recurring routine cards for daily habits and protocols |
| **Drift** | Commitment drift analysis showing how tasks shift over time |
| **Calendar** | Calendar view and time blocking |
| **Time Audit** | Time audit challenge for tracking how you spend your day |
| **Wiki** | Knowledge base for team and personal reference articles |

## Always-Visible Features

Core GTD features cannot be hidden because they are fundamental to the GTD methodology:

- **Do Now** -- Next actions
- **Inbox** -- Capture and process
- **Projects** -- Multi-step outcomes
- **Outline** -- Project overview
- **Waiting For** -- Delegated items
- **Someday/Maybe** -- Deferred ideas
- **Areas** -- Areas of responsibility
- **Horizons** -- Purpose, vision, goals
- **Weekly Review** -- System maintenance
- **Help** -- Documentation

## Admin Configuration

Admins control the default state for each feature:

1. Go to **Settings > Admin > General**
2. Open the **Features** section
3. Use the dropdown for each feature to set one of three states:

| State | Behavior |
|-------|----------|
| **On** | Visible for all users. Users can individually hide it if they choose. |
| **Off by default** | Hidden by default, but users can enable it in their own Settings > Features. |
| **Off** | Hidden for all users. Cannot be overridden. |

## Notes

- Hiding a feature only removes it from navigation. Your data is preserved and will reappear if you re-enable the feature.
- The Features tab in Settings always shows all toggleable features regardless of visibility, so you can always find and re-enable them.
