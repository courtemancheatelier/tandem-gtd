# Projects and Next Actions

Projects and next actions are the two most important organizing concepts in GTD. Understanding the relationship between them is what makes the system work.

## What's a Project?

In GTD, a project is **any desired outcome that requires more than one action step.**

This is a much broader definition than most people use. "Hire a new developer" is a project, but so is "get the car inspected" (because it involves calling for an appointment, driving there, and picking it up). Most people have 30-100 active projects at any given time.

Recognizing something as a project is important because it means you need to track the *outcome* separately from the individual tasks. A task might be "call the mechanic to schedule inspection" — the project is "car inspection completed."

## What's a Next Action?

A next action is the **next physical, visible thing you would do** to move a project forward. Not the whole project. Not a vague intention. The literal next thing.

Bad: "Work on taxes"
Good: "Download W-2 from employer portal"

Bad: "Plan vacation"
Good: "Text Sarah to ask which week in July works"

Bad: "Fix website bug"
Good: "Reproduce the login error on staging server"

The power of next actions is that they're immediately doable. You don't have to think about what to do — you already decided that during clarification. When it's time to work, you just... do the next action.

## The Project-Action Relationship

Every active project should have at least one clear next action. If it doesn't, the project is stuck — and you might not notice until your Weekly Review.

Tandem enforces this automatically through the **cascade engine**:

- In a **sequential project**, only the first incomplete task is the active next action. Complete it, and the next task is automatically promoted.
- In a **parallel project**, all tasks are available as next actions simultaneously.
- In a **single actions list**, each task is independent — it's just a container for related one-offs.

You don't maintain a separate "next actions" list. Tandem derives it from your projects automatically.

## Project Types in Tandem

### Sequential

Tasks are completed in order. Only the first incomplete task appears in your Do Now list. When you complete it, the next one becomes available.

Use this for: most projects, anything with a natural order of steps.

Example — "Renew Passport":
1. Download renewal form from State Department website
2. Get new passport photos at CVS
3. Fill out renewal form
4. Mail form with photos and old passport
5. Track application status online

### Parallel

All tasks are available at once. The order doesn't matter.

Use this for: projects where tasks can be done independently in any order.

Example — "Prepare for house guests":
- Clean guest bedroom
- Buy extra towels
- Stock fridge with breakfast food
- Check that air mattress isn't leaking

### Single Actions

A container for related tasks that don't share a common outcome. Think of it as a folder, not a project.

Use this for: grouping related one-off tasks. "Home maintenance" might hold "replace smoke detector battery," "tighten loose cabinet handle," and "clean gutters" — related by location but not by outcome.

## Sub-Projects

Complex projects can be broken into sub-projects. "Renovate kitchen" might have sub-projects for "Design," "Demolition," "Plumbing," "Electrical," and "Finishing."

In Tandem, sub-projects can be sequential or parallel under their parent:

- **Sequential parent** — sub-projects activate one at a time as each completes
- **Parallel parent** — all sub-projects are active simultaneously

The cascade engine manages all of this automatically.

## Tips

- **Keep project titles outcome-focused.** "Website redesign complete" is better than "Website stuff." You should be able to tell when a project is done.
- **If a project has no next action, it's stuck.** During your Weekly Review, check every project for a clear next action.
- **Don't over-plan.** You don't need to list every task for a project upfront. Just make sure there's always a clear next action. You can add more tasks as the project progresses.
- **Projects complete when the outcome is achieved.** Not when all tasks are done — when the result you wanted is real.
